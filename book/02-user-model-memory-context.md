## 问题

把全部历史对话塞进上下文，既昂贵又不可靠；把每条信息都写成长期记忆，则会制造误解与风险。

## 判断

记忆的核心不是“存下什么”，而是“何时值得触发”。稳定偏好、长期目标和反复出现的协作约定，通常比一次性情绪或临时任务更值得记。

## 方法

将信息分成任务、项目、用户三层。任务层随交付结束；项目层服务当前协作；用户层只保留稳定且经确认的信息。每条长期记忆都应带来源、时间与删除入口。

## 边界

不要把亲密感当作留存手段。主动提醒应服务用户的进度与表达，而非为了证明“我记得你”。

## 清单

- 这条信息会在未来多个任务中改变判断吗？
- 它是否稳定、可解释且得到授权？
- 过期后由谁更新或删除？
