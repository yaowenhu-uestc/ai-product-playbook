## 评估维度

| 维度 | 要问的问题 |
| --- | --- |
| 触发 | 何时应该使用它？ |
| 输入 | 用户需要提供什么？ |
| 流程 | 步骤是否可执行？ |
| 工具 | 是否真正需要工具调用？ |
| 边界 | 什么不该做？ |
| 验证 | 如何判断交付正确？ |
| 维护 | 内容变化后如何更新？ |
| 成本 | 是否比手工更省摩擦？ |

## 最小结构

写清楚目标、适用场景、输入、步骤、输出、验证与限制。未被重复使用的临时脚本或临时 Prompt，不必急着沉淀成 Skill。
