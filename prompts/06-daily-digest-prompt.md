# 每日团队摘要生成 Prompt（压缩纪律 + 署名纪律）

> **第三部 · Prompt · 第39章**
>
> 撰写日期：2026-07-10

把一天的群聊、文档变更、会议纪要压成一份能被 20~30 秒扫完的日报。这个 Prompt 的两个设计重点是**硬性压缩上限**和**不猜署名**——背后的判断见 [团队每日摘要：压缩纪律与署名纪律](../agent/18-daily-digest-compression-attribution.md)。

## 适用场景

- 每日/每周把多源事件流（IM、文档编辑日志、commit、会议纪要链接）合成一份摘要卡；
- 需要摘要短到有人愿意读、准到不伤信任。

## Prompt 模板（脱敏）

```text
你要把团队一天的活动压成一份日报。目标读者用 20~30 秒扫完，不是 5 分钟读。

【硬性上限，超出必须删减】
- 总长度 ≤ 350 字
- 条目 ≤ 8 条
- 单条 ≤ 40 字，只写结论不写过程

【只保留三类内容】
1. 决策：谁定了什么方向/取舍
2. 风险：什么可能出问题、卡在哪
3. 完成信号：什么做完了/上线了
判断标准：这条不知道，明天会不会做错事？不会 → 删掉。
链接堆、闲聊、过程性动作一律不进。

【署名纪律：宁可无名，不可猜名】
- 只用可靠来源标注是谁：带 sender 的消息、带 author 的文档编辑、带 name 的 commit。
- 禁止推断：不从上下文猜"这话大概是谁说的"，不默认会议主持人=提案人，不按角色反推负责人。
- 来源不明确时写"无名"，不要写"很可能是某人"。

【脱敏】
- 一律用昵称/角色，禁止出现真实姓名。

【输入】
- 聊天记录（含 sender）：{chat_log}
- 文档变更（含 author）：{doc_changes}
- 会议纪要链接及解析：{meeting_notes}
- 团队成员昵称表：{roster}

【输出】
按"决策 / 风险 / 完成"三组分节，每条一行，行首带责任人昵称（或"无名"）。
```

## 调优记录

| 现象 | 修正 |
| --- | --- |
| 摘要越写越长、没人读 | 加死物理上限（总字数/条数/单条），逼模型做优先级而非全覆盖 |
| 把一件事安到错的人头上 | 加"只认可靠署名源 + 禁止推断"，来源不明写"无名" |
| 会议纪要抓不到 | 约定"纪要链接甩进指定群"，Prompt 里显式接收解析后的纪要 |
| 真名流出到归档 | 生成阶段用昵称，导出阶段再过一遍脱敏校验 |
