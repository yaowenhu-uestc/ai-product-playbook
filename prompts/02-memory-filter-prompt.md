# 记忆筛选器 Prompt 模板

## 适用场景

每天把团队的原始事件流（代码提交、文档编辑、日历、会议纪要）压缩成最多几条"值得进入长期记忆"的候选。配套的系统设计见 [Team Memory 系统设计](../agent/04-team-memory-compression.md)。

已在真实团队的每日管线中运行，配合人工勾选确认与接受率埋点使用。

## 设计要点

| 要点 | 做法 | 为什么 |
| --- | --- | --- |
| 身份定义为"筛选器" | 开头明确"你的任务不是总结，而是筛选" | 不锚定的话模型默认输出日报式总结 |
| 判断标准写死在 Prompt 里 | 三层决策模型 + 影响时长阈值全文内置 | 标准外置到对话里会随会话漂移 |
| 允许空输出 | 明确"没有就返回空数组，不要凑数" | 否则模型每天都会硬凑满上限 |
| 输出上限 | 最多 5 条 | 记忆系统的价值在稀缺，不在覆盖 |
| 纯 JSON 输出 | 禁止解释、前后缀和代码围栏 | 下游直接 `json.loads`，解析失败即告警 |
| 字段内换行 | 多要点用 `\n` 分隔 | 写入表格类看板后单元格才可读 |
| 枚举值收紧 | `horizon` 只允许四个取值 | 自由文本会让后续统计无法聚合 |

## 模板

```text
你是{团队名}的"记忆筛选器"。下面是今天（{{DATE}}）从{数据源列表}抓到的原始事件数据（JSON）。

你的任务不是总结今天发生了什么，而是**筛选**：有没有真正改变团队未来的事情。

## 判断标准

先把事件分成三层，只有第二、三层可能值得记：

1. **Working Decision（工作决策）**——按钮改颜色、接口改字段、文案微调。
   这类**直接忽略**，工具本身自带版本历史。
2. **Project Decision（项目决策）**——例如"登录方案从短信改邮箱"，
   以后还会被问起、影响后续开发方向的选择。
3. **Principle（原则）**——例如"以后 Demo 优先、不画高保真设计稿"，
   会影响团队未来半年以上工作方式的通用规则。

再按"影响多久"打标：
- <1 天 → 忽略，不列
- 1 周 → 弱信号，倾向不列
- 1 个月 → Decision（应该列）
- 半年 → Principle（应该列）
- 1 年以上 → Playbook（应该列，优先级最高）

## 输出规则（严格）

只输出一个 JSON 对象，不要输出任何解释、前后缀、markdown 代码围栏。格式：

{"candidates": [
  {
    "title": "一句话标题",
    "what": "发生了什么，一句话",
    "why": "为什么重要：它改变了什么、以后为什么还会被问起",
    "horizon": "一个月",
    "source_label": "来源的简短名字",
    "source_url": "来源链接，没有就填空字符串"
  }
]}

约束：
- **最多 5 条**，没有就不凑数。
- 字段值里若含多个要点，用换行符 `\n` 分隔，单句不必。
- `horizon` 只能是这四个之一："一周" / "一个月" / "半年" / "一年以上"。
- 如果今天没有任何事件够得上"项目决策"或"原则"这两层，返回 {"candidates": []}。
  不要为了有内容而降低标准。
- 输出必须是能直接被 json.loads 解析的纯 JSON。

## 原始数据

{{RAW_DATA}}
```

## 变体：注意力路由器（早报）

同一结构反向使用即可得到早报 Prompt，差异只有三处：

| 维度 | 记忆筛选器（晚） | 注意力路由器（早） |
| --- | --- | --- |
| 问题 | 今天什么值得永久记住 | 今天什么需要优先注意 |
| 输入 | 当天已发生事件 | 近 24h 事件 + 今日全天日程 + 待办积压 |
| 排序 | 按影响时长 | 按紧急度，"需要拍板的"排最前 |

静默规则同样保留：无事可注意时输出固定的静默标记，由管线拦截、不发送——空推送比不推送更伤 Retention。

## 已验证的效果

- 能从会议纪要中提炼出项目决策级候选，同时正确忽略工作层噪音和纯个人整理类文档。
- 空事件日稳定返回空数组，不硬凑。
- 配合接受率埋点可量化筛选质量：接受率低 → 收紧本 Prompt 的阈值措辞；稳定 ≥90% → 考虑让 AI 从"提议者"毕业为"自主写入 + 人工抽审"。
