# AI Product Playbook 的结构

> **第一部 · 产品手记 · 第2章**

## 结论

这个仓库应该被当成一个长期作品集，而不是内容归档。它的价值不在于一次性写完，而在于持续更新、持续复盘、持续形成可复用方法。

## Playbook 是什么

Playbook 原意是战术手册。它不是理论教材，而是实践指南：

| 名称 | 更像什么 | 适合程度 |
| --- | --- | --- |
| Handbook | 教材、参考书 | 偏静态 |
| Guide | 入门教程 | 偏一次性 |
| Notes | 个人笔记 | 偏零散 |
| Cookbook | 案例配方 | 偏技巧集合 |
| Playbook | 实践方法、战术手册 | 最适合 |

AI 变化很快，没人能写一本稳定的 AI 教科书。Playbook 更适合表达“这是我目前验证有效的一套实践，以后会继续迭代”。

## 仓库定位

AI Product Playbook 的定位是：

> 一本开源的 AI 产品经理实践手册。

它服务三个目标：

1. 沉淀 AI 产品判断
2. 展示 AI PM 的实践能力
3. 把内容资产从短期平台迁移为长期资产

## 内容结构

| 模块 | 内容 | 产出例子 |
| --- | --- | --- |
| Learning | 学习方法论 | AI 学习路径、输入输出系统 |
| Product Notes | 产品手记 | 任务型 AI 与关系型 AI、Reaction 的下一步 |
| Agent | Agent 实践 | MCP、Memory、Context、Workspace |
| Skills | Skill 沉淀 | 写作 Skill、PPT Skill、调研 Skill |
| Prompts | Prompt 资产 | 产品分析 Prompt、复盘 Prompt |
| Resources | 外部资料 | 书、博客、论文、工具 |

## 与内容平台的关系

同一份内容可以拆成不同形态：

| 平台 | 角色 | 适合内容 |
| --- | --- | --- |
| GitHub | 长期沉淀、作品集、可引用资产 | Playbook、Prompt、Skill、方法论 |
| 公众号 | 完整长文 | 系统思考、产品文章 |
| 小红书 / X | 传播入口 | 核心观点、图文摘要、短判断 |

GitHub 不应该被当成流量平台，而应该被当成可信度平台。

## 为什么 GitHub 适合

GitHub 的优势不是排版，而是：

- 版本管理
- 持续更新记录
- 可协作
- 可引用
- 可 Star
- 能被搜索
- 能证明长期维护

如果一年后这个仓库有 100 篇产品手记、几十个 Prompt 和 Skill、几百次提交，它会比几篇爆文更能证明一个 AI 产品经理的长期能力。

## 更新节奏

| 频率 | 内容 |
| --- | --- |
| 高频 | 产品观察、Prompt、资源链接 |
| 中频 | 产品手记、方法论复盘 |
| 低频 | Playbook 结构调整、核心判断重写 |

## 成功标准

这个仓库的成功，不是短期 Star 数，而是：

1. 是否能持续记录真实判断
2. 是否能让别人快速理解我的 AI 产品观
3. 是否能复用到真实工作中
4. 是否能成为长期作品集
5. 是否能随着实践不断变强
